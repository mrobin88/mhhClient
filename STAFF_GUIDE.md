## Daily Steps

1. Open Admin.
2. Find client by name or phone.
3. If not found, add client (name + phone minimum).
4. Add a case note for each meaningful visit.
5. Set `Note date` if the note is for a past day.
6. Upload documents when available.
7. Save.

Rule: one visit = one note.

## PitStop Workers

1. Staff creates worker account in Admin.
2. Worker logs into Worker portal with phone + PIN.
3. Worker clocks in at start and clocks out at end.
4. Staff verify time in Admin or Reports.

## Manager Reports

1. Log into Admin.
2. Open Reports.
3. Download `Manager package`.
4. Open `START_HERE.html`.
5. If totals look too small, uncheck `Only clients created in activity date range`.

## Quick Fixes

- Cannot log in: retry slowly, then ask supervisor for password reset.
- Cannot find client: search phone without dashes, then last name only.
- Report blocked: log into Admin first, then reopen Reports.
- Worker cannot clock in: confirm account, PIN, and location access on device.
- Forgot yesterday's note: add note now and set `Note date` to yesterday.
- Error page: refresh once, relog once, then send screenshot to supervisor/tech.

## Minimum Required Data

- Client name + phone
- Case note for meaningful visits
- Worker clock in/out for PitStop shifts
- Documents uploaded when received
